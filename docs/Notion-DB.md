# Claude Code에 그대로 붙여넣을 프롬프트 — 기존 프로젝트 소급 등록 (테스트 1개)

이미 진행 중인 프로젝트 1개를 분석해서, Notion의 ① Projects와 ③ Tasks에 자동으로 채워넣는 작업입니다.
"한 개만 먼저 테스트"하는 안전한 방식으로 설계했습니다 — 실행 전에 무엇을 입력할지 미리 보여주고, 사용자 확인을 받은 후에만 실제로 노션에 기록합니다.

---

## ⚠️ 실행 전 준비물

이전에 발급받은 것과 동일한 정보가 필요합니다 (이미 가지고 계시면 재사용하시면 됩니다).

1. Notion API 토큰 (ntn_으로 시작하는 값)
2. ① Projects 데이터베이스의 ID
3. ③ Tasks 데이터베이스의 ID

데이터베이스 ID는, 노션에서 해당 표를 전체 페이지로 열었을 때 주소창의 URL에서 확인할 수 있습니다.
`https://www.notion.so/워크스페이스이름/표이름-32자리영문숫자조합?v=...`
여기서 32자리 영문숫자조합 부분이 데이터베이스 ID입니다.

# 환경 설정
- Notion API 토큰: [여기에 토큰을 붙여넣으세요]
- Projects 데이터베이스 ID: [여기에 ID를 붙여넣으세요]
- Tasks 데이터베이스 ID: [여기에 ID를 붙여넣으세요]
- 분석할 프로젝트 폴더: [여기에 프로젝트의 전체 경로를 붙여넣으세요. 예: /Users/내이름/projects/쇼핑몰]
- notion-client 파이썬 패키지를 사용해줘

# 1단계 — 정보 수집 (아직 노션에 아무것도 쓰지 마)

위 프로젝트 폴더를 분석해서 아래 정보를 모아줘:

## Projects DB에 채울 정보
- 프로젝트명: 폴더 이름 또는 package.json의 name 필드
- 상태: 최근 커밋이 활발하면 "개발중"으로 추정 (확실치 않으면 "개발중"을 기본값으로)
- 시작일: git log의 가장 오래된 커밋 날짜
- 기술 스택: package.json의 dependencies, requirements.txt, 또는 주요 설정 파일에서 추출한 프레임워크/언어 목록
- 사용 라이브러리: package.json dependencies 전체 목록 (devDependencies는 제외)
- 프로젝트 구조: 최상위 2단계 정도의 폴더/파일 트리 (node_modules, .git 등은 제외)
- 깃허브 repo 링크: git remote -v 결과에서 추출
- 한줄 소개: README.md 파일이 있다면 그 첫 문단을 참고해서 한 줄로 요약 (없으면 비워둠)

## Tasks DB에 채울 정보 (커밋 1개당 Task 1개)
git log 전체 이력을 가져와서, 커밋 하나당 다음 정보를 정리해줘:
- Task명: 커밋 메시지의 첫 줄
- 상태: "완료" 고정
- 사용 도구: 알 수 없으면 "기타"로 비워둠
- 커밋 링크: 깃허브에 푸시되어 있다면 해당 커밋의 URL (예: https://github.com/사용자/저장소/commit/커밋해시)
- 변경 파일: 그 커밋에서 변경된 파일 목록
- 작업 유형: 커밋 메시지가 "feat:", "fix:", "refactor:" 등으로 시작하면 그 prefix를 추출, 없으면 비워둠
- 완료 시각: 그 커밋의 날짜/시각

커밋 개수가 너무 많으면(50개 이상) 전부 가져오지 말고, 최근 30개만 우선 가져오고 "전체 커밋은 OO개이며 최근 30개만 가져왔습니다"라고 알려줘.

# 2단계 — 미리보기 보여주기 (반드시 이 단계를 거쳐야 함)

노션에 쓰기 전에, 위에서 수집한 정보를 보기 좋게 정리해서 화면에 보여줘:
- Projects에 들어갈 내용 1건을 표로 보여주고
- Tasks에 들어갈 내용을 몇 건인지 개수와 함께, 처음 3건만 예시로 표로 보여줘
- 그리고 "이대로 노션에 등록할까요? (네/아니오)"라고 물어봐줘

# 3단계 — 사용자가 "네"라고 확인한 경우에만 실제로 Notion API를 호출해서 등록해줘

등록 순서:
1. Projects DB에 프로젝트 1건 생성
2. 방금 생성된 프로젝트의 페이지 ID를 가져와서, Tasks DB에 각 Task를 생성할 때 "프로젝트" relation 속성에 그 ID로 연결
3. 다 등록한 후 "Projects 1건, Tasks OO건 등록 완료"라고 알려줘

# 주의사항
- 절대 2단계 미리보기 없이 바로 노션에 쓰지 마
- 만약 같은 프로젝트명이 Projects DB에 이미 있다면, 중복 생성하지 말고 나에게 먼저 알려줘
- API 토큰은 코드에 직접 박지 말고 .env 파일로 분리해줘 (이미 .env가 있다면 그걸 재사용)

## 사용 방법

1. 위 프롬프트의 대괄호 `[ ]` 부분 4곳을 실제 값으로 채워주세요
2. 테스트할 프로젝트 폴더로 Claude Code를 열어서 (또는 터미널에서 그 폴더로 이동한 뒤) 프롬프트를 붙여넣으세요
3. 1, 2단계가 끝나면 화면에 "이대로 등록할까요?"라는 미리보기가 나옵니다 — 내용을 꼼꼼히 확인하세요
4. 이상 없으면 "네"라고 답하면 그때 노션에 실제로 등록됩니다

## 테스트 후 확인할 것

- 노션 ① Projects에 프로젝트가 1건 잘 들어갔는지
- ③ Tasks에 커밋 개수만큼 행이 들어갔는지, "프로젝트" 칸이 ①과 잘 연결(relation)되어 있는지
- 날짜, 커밋 링크 등이 실제 깃허브 정보와 맞는지

문제없이 확인되면, 같은 프롬프트를 다른 프로젝트 폴더 경로로 바꿔가며 나머지 프로젝트에도 반복 적용하시면 됩니다.
